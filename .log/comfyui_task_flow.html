ComfyUI 任务状态判断与反馈流程

核心函数：queue_prompt + check_status
SUCCEEDED FAILED PENDING RUNNING

5. 树形判断结构（check_status）

check_status(prompt_id, server_address)
│
├─ 选择待检查服务器 servers_to_check
│   ├─ 如果 server_address 非空 → [仅此一台]
│   └─ 否则 → [遍历 SERVER_LIST 中所有服务器]
│
└─ 对每个 server in servers_to_check:
    │
    ├─ history = get_history(prompt_id, server)
    │   ├─ history is None
    │   │   └─ any_network_error = True          # 不立即失败，只记录网络异常
    │   │
    │   └─ history 中包含 prompt_id
    │       ├─ 解析 outputs，选出最佳文件
    │       └─ 返回  SUCCEEDED, file_info        # 整个函数结束
    │
    └─ status = is_task_running(prompt_id, server)
        │
        ├─ status == "UNKNOWN"
        │   └─ any_network_error = True          # 视为网络异常
        │
        ├─ status ∈ {"PENDING", "RUNNING"}
        │   └─ 返回  status, None                # 任务仍在排队/运行
        │
        └─ status == "NOT_FOUND"
            │
            └─ 如果刚才 history 非 None:
                ├─ 再次调用 get_history(prompt_id, server)
                │   ├─ 若找到 → 递归调用 check_status(prompt_id, server)
                │   └─ 若返回 None → any_network_error = True
                └─ 否则继续下一台 server

遍历结束后的总决策：

├─ 如果在任一 server 上返回 SUCCEEDED
│   └─ 已在循环中提前返回
│
├─ 如果 any_network_error 为 True
│   └─ 返回  PENDING, None
│       # 有网络抖动，保持任务挂起状态，避免误判 FAILED
│
└─ 否则（所有服务器都正常返回，且都没有该任务）
    └─ 返回  FAILED, "Task not found"

6. 思维导图：成功路径与失败路径

ComfyUI 任务状态判断（思维导图）

                      ┌──────────────────────────┐
                      │   ComfyUI 任务状态判断   │
                      └──────────┬──────────────┘
                                 │
        ┌────────────────────────┼────────────────────────┐
        │                        │                        │
        v                        v                        v
  成功路径（SUCCEEDED）    挂起/进行中路径（PENDING/RUNNING）   失败路径（FAILED）


成功路径（SUCCEEDED）
---------------------
- 选择服务器：
  - 使用 server_address 或 SERVER_LIST
- 对某一台 server：
  - get_history(prompt_id, server)
    - 找到包含该 prompt_id 的记录
      - 解析 outputs，收集所有节点输出文件
      - 选择最佳 file_info（优先视频，然后图片/音频）
      - 立即返回：
        - status = "SUCCEEDED"
        - result = file_info
- app 层的后续动作：
  - 视频分段：
    - 下载到本地 result_path
    - 记录 task['status'] = 'completed'
    - 最后拼接视频、合并音频、上传 OBS
  - 音频任务：
    - 下载音频，转 WAV，上传 OBS
    - 更新 AUDIO_TASKS 中的 url
    - 触发数字人视频生成


挂起/进行中路径（PENDING / RUNNING）
-----------------------------------
- 某台服务器上 history 为 None 或 queue 接口异常：
  - any_network_error = True
- 某台服务器上队列中存在该任务：
  - status ∈ {"PENDING", "RUNNING"}
  - 返回 ("PENDING" or "RUNNING", None)
- 所有服务器遍历完成后：
  - 若 any_network_error 为 True 且没有成功结果：
    - 返回 ("PENDING", None)
    - 含义：网络抖动或正在执行，保持挂起，稍后重试


失败路径（FAILED）
------------------
- 所有服务器都被正常访问：
  - 对每台 server：
    - get_history(prompt_id, server) 返回空或不含该 id
    - queue 中也没有该 prompt_id（NOT_FOUND）
    - 没有出现 UNKNOWN 或异常（即 any_network_error 为 False）
- 汇总判断：
  - 没有任何 SUCCEEDED
  - 没有任何网络错误
  - 任务在所有服务器的 history/queue 中都不存在
- 最终返回：
  - status = "FAILED"
  - result = "Task not found"
- app 层的后续动作：
  - 标记任务或任务组为 failed
  - 记录 error 信息，用于对外返回或日志排查

1. 提交任务 queue_prompt

构造请求
queue_prompt(prompt)
包装为 JSON：
{"prompt": workflow, "client_id": ...}
⟶
发送到 ComfyUI
POST /prompt
成功返回：
{"prompt_id": "..."}
⟶
记录任务
app 中保存：
(prompt_id, server_address)
写入 TASKS_STORE / AUDIO_TASKS

2. 任务轮询 check_status

选择服务器
入参：
check_status(prompt_id, server_address=None)
逻辑：
- 有 server_address：只查这一台
- 否则遍历 SERVER_LIST
⟶
查询 history
GET /history/<prompt_id>
- 有数据且包含该 id：
 解析 outputs，选出最佳文件
 返回 SUCCEEDED
- 返回 None：记为网络错误
⟶
查询 queue
GET /queue
- 在 pending/running 队列里：
 RUNNING 或 PENDING
- 状态 UNKNOWN：
 记为网络错误
- NOT_FOUND 且刚查过 history：
 再试一次 history

3. 最终状态判定

有任一服务器 SUCCEEDED
立即返回：
("SUCCEEDED", file_info)
上层逻辑：
- 下载结果文件
- 拼接视频 / 上传 OBS
⟶
有网络错误
所有服务器都查完后：
- 只要出现过网络异常
 返回 PENDING
- 避免把网络抖动当成任务 FAILED
⟶
无输出且无网络错误
条件：
- history 无该 id
- queue 中也不存在
- 没有网络异常
最终：
返回 FAILED
("FAILED", "Task not found")

4. app 层如何使用状态

视频分段任务
循环各段：
status, result = check_status(task_id, server)
- SUCCEEDED: 下载到 result_path，标记 completed
- FAILED: 标记 failed，记录 error
- PENDING/RUNNING: 保持未完成状态
⟶
音频任务
轮询：
check_status(prompt_id, server)
- SUCCEEDED: 下载音频，转 wav，上传 OBS
- 然后触发数字人视频生成
⟶
对外接口反馈
HTTP 接口返回：
- group: status + final_url
- audio: status + url
状态直接映射：
- completed / failed / processing
